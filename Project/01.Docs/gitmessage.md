## 소제목

```txt
# 제목은 최대 50글자까지 아래에 작성: ex) Feat: Add Key mapping

# 본문은 아래에 작성

# 꼬릿말은 아래에 작성: ex) Github issue #23

# --- COMMIT END ---
#   <타입> 리스트
#   Feat        : 새로운 기능 추가
#   Fix         : 버그 수정
#   Bug         : 버그 발생
#   Refactor    : 코드 리팩토링
#   MarkUp      : 마크업 (HTML) 변경
#   Design      : 사용자 UI 디자인 변경
#   Style       : 코드 형식, 세미콜론 추가 등 스타일 변경
#   Docs        : 문서 수정 (문서 추가, 수정, 삭제, README)
#   Init        : 초기 생성, 라이브러리, 프레임워크 등 설치
#   Rename      : 파일 혹은 폴더명을 수정하거나 옮기는 작업만 한 경우
#   Remove      : 파일을 삭제하는 작업만 수행한 경우
#   Sync        : 팀원들과 동기화
#   Deploy      : 배포
# ------------------
#   1. commit 다같이 모였을 때 합니다!
#   2. 명령문으로 이해하기 쉽게 작성 합니다!
#   3. 키워드 순서대로 내용도 순서대로 일치하게 작성 합니다 !
#   ex) Feat, Docs : MyScedulePage div 4개 추가(feat 내용), React Calender 라이브러리 설치(docs 내용)
# ------------------
#   <꼬리말>
#   필수가 아닌 optioanl
#   Fixes        :이슈 수정중 (아직 해결되지 않은 경우)
#   Resolves     : 이슈 해결했을 때 사용
#   Ref          : 참고할 이슈가 있을 때 사용
#   Related to   : 해당 커밋에 관련된 이슈번호 (아직 해결되지 않은 경우)
#   ex) Fixes: #47 Related to: #32, #21
```
